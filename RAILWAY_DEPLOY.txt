ATHLETEFORGE — DEPLOY ON RAILWAY (PRIMARY)
============================================
Public URL like: https://athleteforge-bca-production.up.railway.app
Time: ~10 minutes (one-time)

STEP 1 — Account
  https://railway.com → Sign in with GitHub (rohith917)

STEP 2 — New project from GitHub
  New Project → Deploy from GitHub repo
  Select: athleteforge-bca
  Branch: main
  Railway reads railway.toml + Dockerfile automatically (no Root Directory!)

STEP 3 — PostgreSQL database
  In project: + New → Database → PostgreSQL
  Wait until status = Active

STEP 4 — Link database to web service (REQUIRED — service stays offline without this)
  Click the WEB service (not Postgres)
  Variables tab → + New Variable → Add Reference
  Pick: Postgres → DATABASE_URL
  You must see DATABASE_URL listed on the web service variables.

STEP 5 — Web service variables (Variables tab)
  SECRET_KEY         = athleteforge-bca-secret-2026-change-me
  DEBUG              = False
  SAME_ORIGIN_DEPLOY = True

  (Railway auto-sets PORT and RAILWAY_PUBLIC_DOMAIN — do not override)

STEP 6 — Public domain
  Web service → Settings → Networking → Generate Domain
  Copy URL, e.g. https://athleteforge-bca-production.up.railway.app

STEP 7 — Deploy
  Deployments tab → wait for Success (first build ~5-10 min)
  Health check: /api/health/

STEP 8 — Open your site
  Paste the .railway.app URL in your browser (do not Google-search it)

TROUBLESHOOTING — "Service offline"
  See: RAILWAY_FIX_OFFLINE.txt
  #1 fix: DATABASE_URL reference missing on web service → add it → Redeploy
  Build failed?     → Deployments → View logs → Redeploy
  502 on first load → Wait 2-3 min (migrations on first boot)
  White screen?     → Hard refresh (Ctrl+Shift+R)
